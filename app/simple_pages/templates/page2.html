Docker is a tool that allows developers to easily deploy their applications in a containers to run on the host operating system (like Linux).

Here are some key terms to know when thinking about Docker:

  • Containers - a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.
  • Images - The blueprints of an application which form the basis of containers.
  • Docker Hub - A registry of Docker images. Think of the registry as a directory of all available Docker images.

Keeping this terminology in mind, we can begin working with Docker. Like Git, Docker shares the concept of a repository, pushes, and pulls and comes with a bunch of useful commands. Repositories (which can be created in Docker Hub by clicking on Repositories and then Create Repository) allow you to share container images with others through the Docker community. Docker images can be pushed to Docker hub with the docker push command. To connect to Docker when starting a project in PyCharm, go to Services, and then add and start Docker. Then use the docker compose up --build command to build the container. In the terminal (not PyCharm) first use docker ps command to get the name of the existing container. Then, use docker exec -it [container name] /bin/bash to get a bash shell in the container. From there you can use docker exec -it [container name] [command] to execute any command you want in the container. Below is a list of useful Docker commands to help you!

Useful Docker commands:

  • create — Create a container from an image.
  • start — Start an existing container.
  • ls — List running containers.
  • rm— Delete a stopped container.
  • build — Build an image.
  • push — Push an image to a remote registry.
