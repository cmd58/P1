CI and CD stand for continuous integration and continuous delivery/continuous deployment.

Here are some key terms to remember when thinking about CI/CD:

CI - a software development practice in which incremental code changes are made frequently and reliably.

CD - a software development methodology that constitutes making updates continuously, piece-by-piece, enabling software code to be delivered to customers as soon as it is completed and tested.

So what is meant CI/CD as a whole?

Continuous Integration is the practice of testing each change done to your codebase automatically and as early as possible. Continuous Deployment follows the testing that happens during Continuous Integration and pushes changes to a staging or production system.

And what is GitHub Actions

GitHub Actions is an automation tool which gives developers CI/CD capabilities right next to your code in GitHub.
