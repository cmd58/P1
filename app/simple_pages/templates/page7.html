Object Oriented Programming is a programming “paradigm” based on classes and objects. It aims to create reusable pieces of code (classes) to prevent being redundant in your program.

OOP Glossary:

object - instances of a class created with specifically defined data.

class - user-defined data types that act as the blueprint for individual objects, attributes and methods.

namespace - a declarative region that provides a scope to the identifiers (the names of types, functions, variables, etc) inside it.

constructor - a special kind of method that Python calls when it instantiates an object using the definitions found in your class.

fixture - define the steps and data that constitute the arrange phase of a test.

type hint - a way to make Python feel statically typed giving developers a way to make their code as self-describing as possible.

type cast - also known as "explicit type conversion" the data types of objects are converted using predefined functions by the user.

unit test - a software testing method by which individual units of source code are put under various tests to determine whether they are fit for use.

static variable - a variable that has been allocated "statically", meaning that its lifetime (or "extent") is the entire run of the program.

static method - a method defined as a member of an object but is accessible directly from an API object's constructor, rather than from an object instance created via the constructor.

instance method - methods which require an object of its class to be created before it can be called.

instance property - a Python variable belonging to only one object.

instantiation - creating a copy of the class which inherits all class variables and methods.

encapsulation - the process of wrapping up variables and methods into a single entity.

inheritance - existing classes can be modified by a new class.

polymorphism - refers to the use of a single type entity (method, operator or object) to represent different types in different scenarios.

abstraction - a process of handling complexity by hiding unnecessary information from the user.

How the Calculator Program Demonstrates OOP

Encapsulation - The implementation and state of each object which are the calculators operations (add, subtract, multiply) are privately held inside a defined class which is the Calculator class.

Inheritance - Classes can reuse code from other classes. In Python, every class inherits from a built-in basic class called 'object'. In the case of the calculator program the constructor (the '__init__') function of the Calculator class is invoked when we create an object variable or an instance of the class.

Polymorphism - Child class that have the same name as the methods in the parent class. This can be seen in the calculator_test.py where we import the calculator class as the default result property for the new Calculator class "calculator".

Abstraction - Objects only reveal internal mechanisms that are relevant for the use of other objects, hiding any unnecessary implementation code. This is implemented by using "self" throughout the calculator program.
