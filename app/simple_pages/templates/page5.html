Pylint is a tool that checks for errors in Python code and then gives the code an overall mark based on the number and severity of warnings and/or errors.

Installing Pylint

Install pylint with the command: pip install pylint

How Pylint Works

Pylint is a static code analysis tool that is meant to be called from the command line, where you can give it a python package(s) or module(s) to analyze. Given a file pylint will attempt to convert it to a module that it can analyze instead. Pylint uses both unittests and functional tests. To use pylint, simply use the command: pylint (filename/modulename).py

The Output of Pylint

Pylint returns an output of semantic, syntax, and style errors as well as bugs and redundant code warnings. It also returns a score or mark that lets us know how ideal the given code is.

Why Use Pylint?

When coding in Python, you do not have a compiler to catch bugs for you, so that is where pylint comes in. It acts as a lint tool that catches the bugs in your code. There are others like pylint such as pychecker. pyflakes, and flake8.
