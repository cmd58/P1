AAA Testing is a suggested testing concept that says tests should be divided into three sections, Arrange, Act, and Assert.

Tutorial with Python

The “Arrange-Act-Assert” (also AAA and 3A) pattern of testing focuses each test on a single action. The advantage of this focus is that it clearly separates the arrangement of the System Under Test (SUT) and the assertions that are made on it after the action. Check out an example of a full AAA Test code to the right. The pattern looks like this:

  • Test definition, docstring, and Arrangement
  • Empty Line
  • Single line of code, the Act
  • Empty Line
  • Assert block code or the Assertions

So what do all of these pieces mean and more importantly how do we use them?

Follow these steps for an AAA compliant test!

  1. Define the test function with a meaningful name (as it will be used if the test fails in the output).
  2. Optional: add a short single line docstring about the test.
  3. Arrange - Using a single block of code with no empty lines, set up the conditions for the test action. Do NOT use assert.
  4. Act - This is the line of code where the action is taken on the SUT. Each line should start with result =.
  5. Assert - This is the block of code that performs assertions on the state of the SUT after the action. Use a single block of code with no empty lines. First test result, then side effects.
